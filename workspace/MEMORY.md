## 产品决策记录

### 知识架构：符号化知识编译（非 RAG）

**决策**：使用结构化 Markdown + 图遍历替代传统向量库 RAG。
- **原因**：MCN 知识量有限且结构化，知识价值在于关联性（达人与品牌、会议决策与后续执行）
- **方案**：ingest → knowledge/wiki/sources/ + entities/ + concepts/ → build_graph.py → query.py 图增强检索
- **状态**：✅ 已落地（wiki_ingest.py / wiki_query.py / build_graph.py）

### 风险检测：嵌入式预警（非独立告警）

**决策**：风险只做周报中的 ⚠️ section，不做独立实时推送。
- **原因**：误报代价大，独立告警会导致"狼来了"效应
- **方案**：数据规则 + 关键词匹配 + LLM 综合判断，阈值由场景配置决定
- **状态**：✅ 已定义（weekly-reporter 内部嵌入检测，阈值配置 config/scenarios/mcn.yaml）

### 场景通用化

**决策**：换场景 = 换 `config/scenarios/*.yaml` + 换 `prompts/*.md`，代码零改动。
- **当前**：MCN 场景已配置（mcn.yaml）
- **验证**：待接入第二个场景（候选：内容创作部 / 软件研发部）
- **状态**：⏳ 初赛聚焦 MCN，复赛考虑跨场景验证
