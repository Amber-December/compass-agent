# intent-classifier

Q&A Bot 专用的意图识别能力。

## 能力

- 识别用户查询意图类别
- 判断是否需要拒答（跨项目敏感数据）
- 提取查询中的关键实体（项目名、人名、时间范围）

## 意图分类

| 意图 | 说明 | 处理方式 |
|------|------|----------|
| `data_query` | 查询项目 KPI/进度数据 | 检索 Base 数据 |
| `knowledge_query` | 查询 SOP/方法论/经验 | 检索 Wiki/妙记 |
| `cross_project_shallow` | 跨项目浅查询（部门级聚合） | 可回答，只给汇总 |
| `cross_project_deep` | 跨项目深查询（其他项目明细） | 拒答，引导到对应群 |
| `boundary_extrapolation` | 数据外的外推/预测 | 拒答，明确说无数据 |
| `meta_info` | 查询项目列表/PM/组织架构 | 可回答 |

## 拒答规则

以下情况必须拒答：
- 查询其他项目的 KPI 具体数值（GMV、ROI、复购率明细）
- 查询其他项目的达人个人绩效、违规记录
- 查询其他项目的整改进度细节
- 要求对无数据的情况进行预测/外推

拒答话术模板：
```
"⚠️ 跨项目敏感数据未开放，如需 [项目名] 的详情，请到该项目群内询问。"
```
