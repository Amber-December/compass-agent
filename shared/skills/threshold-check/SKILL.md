# threshold-check

风险检测与阈值判断能力，用于识别项目异常指标。

## 能力

- 根据场景配置中的阈值规则判断 KPI 是否异常
- 支持多种判断逻辑（绝对值、环比、连续 N 期）
- 输出风险等级和具体触发原因

## 阈值规则类型

| 类型 | 说明 | 示例 |
|------|------|------|
| `absolute_lt` | 绝对值小于阈值 | GMV < 40万 |
| `absolute_gt` | 绝对值大于阈值 | 故障率 > 5% |
| `consecutive_decline` | 连续 N 期下降 | GMV 连续 2 周下滑 |
| `consecutive_below` | 连续 N 期低于阈值 | 粉丝增长率连续 2 月 < 5% |
| `rate_change` | 变化率超过阈值 | 环比下滑 > 30% |

## 风险等级

- **🔴 高风险**：核心业务指标严重偏离，需立即干预
- **🟡 中风险**：指标异常，需关注和跟进
- **🟢 正常**：指标在合理范围内

## 配置方式

阈值定义在 `config/scenarios/<name>.yaml` 中，按项目定制。
