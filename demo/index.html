02
系统架构 System Architecture

Compass 不是单一聊天机器人，而是基于 OpenClaw 的多 Agent 协作系统。

设计哲学 · 为什么不是单体 AI？

单体 AI 的局限

职责混杂导致上下文爆炸 · 无法独立演进 · 单点故障风险高 · 权限边界模糊

多 Agent 的优势

单一职责 Prompt 更聚焦 · 独立部署可单独升级 · 能力可组合复用 · 权限物理隔离

完整调用链路 · 从消息到回答的 10 步流程

Step 1-2 · 消息接收

用户在"垂类达人孵化项目群"发送 "@司南 达人筛选标准是什么" → Gateway 接收

Step 3 · Scope 绑定

chatBindings 查找 chat_id → scope = "kol-incubation"

Step 4-5 · 意图识别

qa-bot intent skill 分析 → "知识查询" / 置信度 0.95 / target_skill = "query"

Step 6-7 · 知识检索

query skill 调用 wiki_query.py → 读取 wiki/ 目录 → 按 scope 过滤 → 匹配 "达人筛选标准.md"

Step 8 · LLM 润色

LLM 基于检索结果 + SKILL.md 模板生成自然语言回答

Step 9-10 · 输出推送

红线检查（无思考过程 / 无本地路径 / 只一条消息）→ Gateway 推送到飞书群聊

03
核心创新 Key Innovations
01

Scope-Aware 权限隔离

根据聊天群自动推断项目范围，无需登录。更精细的是数据分级：PM 姓名等公开信息可跨项目回答，GMV/转化率等敏感 KPI 严格按 scope 拒答。

群即边界
02

模糊匹配 + 默认本项目

用户无需记忆指令格式。"本周达人变现多少"自动识别当前 scope；"看看本周数据"返回最新 KPI。像和同事聊天一样自然。

零学习成本
03

数据查询自动降级

数据查不到时，不编造、不外推，而是自动切到文档查询，返回相关 SOP 链接，告知最新数据时间范围。

Honest AI
04

交互式知识图谱子图

从全量图谱提取关键词子图（含 1 跳邻居），生成 vis.js 可交互 HTML。可拖拽、缩放、查看节点详情，不是静态截图。

可视化的知识
05

严格输出约束（Red Lines）

通过 SKILL.md 强制规则确保 LLM 输出"像人话"：

✗ 输出思考过程 · ✗ 暴露本地路径 · ✗ 描述检索过程 · ✗ 分条发送

✓ 只输出一条最终消息 · ✓ 数据来源可溯源 · ✓ 附带飞书原文链接

高阶 AI 技巧

意图分类 + 路由分发

非端到端生成。先分类（文档/数据/图谱/闲聊），再路由到对应 Skill。降低幻觉，提升可控性。

Prompt 工程三层架构

systemPrompt（全局行为）→ IDENTITY.md（人格）→ SKILL.md（具体能力）。LLM 在结构化模板中填空。

RAG + 结构化查询双轨

文档层用 RAG 检索非结构化知识；数据层用结构化过滤精确数值。两者互补，覆盖 90%+ 场景。

知识图谱增强

构建异构知识图谱（项目-文档-达人-指标），支持全局可视化、子图提取、隐性关联发现。

可复用性与推广价值

Compass 架构不绑定 MCN 行业。任何具备多项目并行 + 文档/数据双轨知识 + 权限隔离需求 + 群聊协作入口的团队均可复用。

潜在场景：律所（案件知识库）、咨询公司（项目交付物管理）、研发管理（多团队文档+指标）、医疗（科室知识+病例数据）。
