# Compass（司南）qa-bot 能力演示脚本

> 本脚本用于演示 qa-bot 在多项目群场景下的完整能力设计，覆盖权限隔离、模糊匹配、知识/数据/图谱检索、自我介绍、闲聊拒答等核心功能。
>
> 演示方式：在真实项目群（或测试环境）中按顺序发送消息，观察 qa-bot 的回复行为。

## 第一幕：垂类达人孵化项目群 — 自己项目能力展示

> **场景**：用户在垂类达人孵化项目群（scope = `kol-incubation`）提问，qa-bot 应默认将模糊查询指向本项目。

---

### 1.1 自我介绍

**输入**：
```
@司南 你是谁
```

**预期回复**（必须包含）：
- 开场白：`👋 你好！我是 **司南**，团队效能小助手，目前在 **垂类达人孵化项目群**~ 🧭`
- 能力列表中的例子与垂类达人孵化相关（达人、签约、商单、SOP）
- 结尾引导：`"达人筛选标准是什么"`、`"看看本周数据"`

**验证点**：
- [ ] 群名称正确显示"垂类达人孵化项目群"
- [ ] 只输出一条消息，无思考过程
- [ ] 例子与当前项目相关（非通用例子）

---

### 1.2 文档检索 — 自己项目 SOP（模糊匹配）

**输入**：
```
@司南 达人筛选标准是什么
```

**预期行为**：
- 模糊匹配识别：未指定项目名称 → 默认检索 `kol-incubation`
- 检索 `wiki/concepts/达人筛选SOP.md`
- 正常回答三维度评分模型及权重

**预期回复**：
```markdown
**回答**：垂类达人孵化的达人筛选采用三维度评分模型。

**详细说明**：
- 基础素质 40%（形象气质、表达能力、专业背景）
- 内容潜力 35%（创意能力、专业度、差异化）
- 商业潜力 25%（配合度、粉丝画像、变现能力）

**来源**：
- [[达人筛选SOP]] — 三维度评分模型定义

**飞书原文**：
- [达人筛选SOP](https://ncnkdep1f4r7.feishu.cn/wiki/...)
```

**验证点**：
- [ ] 未要求用户重复指定"垂类达人孵化"
- [ ] 包含 `[[PageName]]` 引用
- [ ] 包含飞书 Wiki 原文链接
- [ ] 无思考过程、无本地路径暴露

---

### 1.3 数据检索 — 自己项目 KPI（模糊匹配）

**输入**：
```
@司南 本周达人周均变现多少？商单转化率多少？
```

**预期行为**：
- 模糊匹配：未指定项目名称 → 默认读取 `kol-incubation` Week 9 数据
- 查询飞书 Base — 垂类达人孵化项目周度KPI

**预期回复**：
```markdown
**查询结果**：

| 指标 | Week 9 数值 | 目标值 | 状态 |
|------|------------|--------|------|
| 达人周均变现 | 27,167 元 | — | — |
| 商单转化率 | 20% | — | — |

**数据来源**：[飞书 Base — 垂类达人孵化](https://ncnkdep1f4r7.feishu.cn/base/LTLNbBlGTaB4mNs6by0cvhPhnib)
**时间范围**：2026-04-27 至 2026-05-03（Week 9）
```

**验证点**：
- [ ] 数值正确（27,167 / 20%）
- [ ] 数据来源为飞书 Base 链接，非本地路径
- [ ] 时间范围明确
- [ ] 无思考过程、无"扫描文件"等检索描述

---

### 1.4 知识图谱检索 — 自己项目子图

**输入**：
```
@司南 看看垂类达人孵化的知识图谱
```

**预期行为**：
- 意图识别为图谱查询
- 调用 `graph_query.py` 搜索"垂类达人孵化"子图
- 返回子图描述 + HTML 文件路径/服务地址

**预期回复**：
```markdown
**"垂类达人孵化" 子图搜索结果**：
- 直接匹配节点数：2
- 子图总节点数：53（含 1 跳邻居）
- 子图边数：194

**直接匹配节点**：
- 垂类达人孵化（类型: source）— 项目文档
- 垂类达人孵化（类型: entity）— 项目实体

**交互式子图**：
- 已生成 HTML：workspace/knowledge/graph/subgraph_垂类达人孵化.html
- 本地打开查看或联系管理员获取访问权限
```

**验证点**：
- [ ] 返回结构化子图信息
- [ ] 节点数和边数统计正确
- [ ] 提供查看方式


**预期行为**：
- 明确指定跨项目"直播电商"

**预期回复**：

### 1.6 跨项目敏感问题 — KPI 数据（应拒答）

**输入**：
```
@司南 直播电商上周总 GMV 是多少？直播转化率多少？
```

**预期行为**：
- 明确指定跨项目"直播电商"
- 问题涉及**具体 KPI 数值**（GMV、转化率）
- 当前 scope = `kol-incubation`，目标项目 = `live-commerce`
- **必须拒答**

**预期回复**：
```markdown
⚠️ 跨项目敏感数据未开放，如需直播电商的详情，请到该项目群内询问。
```

**验证点**：
- [ ] 明确拒答，不透露 GMV（487,500）和转化率（4.2%）
- [ ] 使用标准拒答话术
- [ ] 给出正确引导（到该项目群询问）
- [ ] ❌ 错误：回答数值后附加提醒（已泄露）

## 第二幕：品牌代运营项目群 — 切换 scope 后的对比

> **场景**：同一个用户切换到品牌代运营项目群（scope = `brand-ops`），询问之前在垂类达人孵化群被拒答的敏感问题。此时该问题变为"自己项目"问题，应正常回答。

---

### 2.1 自我介绍（群切换感知）

**输入**：
```
@司南 你是谁
```

**预期回复**（必须包含）：
- 开场白：`👋 你好！我是 **司南**，团队效能小助手，目前在 **品牌代运营项目群**~ 🧭`
- 能力列表中的例子与品牌代运营相关（品牌、客户、曝光量、满意度）

**验证点**：
- [ ] 群名称已切换为"品牌代运营项目群"
- [ ] 例子与品牌代运营项目相关

---

### 2.2 同样的问题，现在可以回答 — 自己项目 KPI

**输入**：

- **正常回答**

**预期回复**：

```bash
  -H "Content-Type: application/json" \

    "scope": "kol-incubation",

  }'

# 批量运行演示用例
python eval/run_demo.py --script eval/qa_bot_demo_script.md
```
