# 周报撰写与复盘规范
> MCN 事业部项目周报与复盘文档的标准格式与撰写要求，适用于全体 PM 和项目成员。

---

## 一、周报结构

### 1.1 标准模板
```markdown
# [项目名称] 周报 [开始日期] 至 [结束日期]

## 1. 本周数据看板
| 指标 | 本周 | 上周 | 环比 | 目标 | 状态 |
|------|------|------|------|------|------|

## 2. 风险与阻塞
- 项目整体状态：[🟢正常 / 🟡预警 / 🔴风险]
- 触发原因：...
- 其他风险：...

## 3. 本周亮点
- ✅ ...

## 4. 下周计划
- ...

## 5. 相关链接
- [项目 OnePage](...)

```

### 1.2 数据看板要求

- **必须包含**：核心KPI（不超过5个）
- **环比计算**：`(本周 - 上周) / 上周 * 100%`，保留1位小数
- **状态标记**：
  - 🟢 达标（≥100%目标）
  - 🟡 预警（80%-100%目标）
  - 🔴 风险（<80%目标）

---

## 二、复盘规范

### 2.1 周复盘会（每周五）

**参会人**：项目全员**输出物**：会议纪要 + action items**议程**：

1. 数据回顾（5min）：核心KPI达成情况
1. 亮点分享（10min）：本周做得好的，可复制
1. 问题分析（15min）：未达标指标的原因与归因
1. 行动计划（10min）：下周改进措施，明确owner和deadline

### 2.2 复盘文档模板
```markdown
# [日期] 项目复盘

## 1. 目标回顾
- 原定目标：...
- 实际结果：...

## 2. 过程分析
- 做得好的（Keep）：...
- 需改进的（Problem）：...
- 新尝试的（Try）：...

## 3. 根因分析（5 Whys）
问题：... 
为什么？→ ...
为什么？→ ...
（追问至第5层）

## 4. 行动计划
| 行动项 | Owner | Deadline | 验收标准 |
|--------|-------|----------|----------|

```

---

## 三、撰写要求

### 3.1 数据准确性

- 周报数据必须与飞书 Base 一致
- 如有数据修正，需说明修正原因
- 禁止编造或估算数据

### 3.2 语言规范

- 使用客观描述，避免情绪化表达
- 问题归因具体，避免"各种原因""大概"
- 行动计划可执行，避免"加强""优化"等模糊动词

### 3.3 时效性

- 周报：每周日24:00前归档至Wiki
- 会议纪要：会后24h内归档
- 复盘文档：复盘会后48h内完成

## 四、示例：数据洞察写法

**❌ 错误示例**：
> 本周GMV下降了，可能是因为投流效果不好。

**✅ 正确示例**：
> 【数据事实】本周 GMV 16.25 万，环比下降 4.4%【原因分析】投流 ROI 从 1:2.7 下滑至 1:2.5，千川人群包老化导致流量质量下降【影响评估】连续 2 周下滑，距离目标 20 万差距扩大【行动计划】施十六本周完成投流策略复盘，更新3个人群包，目标ROI回升至1:3
