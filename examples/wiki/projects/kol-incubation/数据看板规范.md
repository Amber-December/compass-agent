# 数据看板规范

## 一、看板架构

### 1.1 数据层级
```plaintext
项目总览层 → 达人个体层 → 内容条目层

```


| 层级 | 更新频率 | 查看权限 | 主要用途 |
| --- | --- | --- | --- |
| 项目总览层 | 实时 | 全员 | 项目整体健康度 |
| 达人个体层 | 每日 | PM+运营 | 达人成长追踪 |
| 内容条目层 | 实时 | 运营+达人 | 单条内容效果分析 |

### 1.2 数据来源

| 数据类型 | 来源 | 同步方式 |
| --- | --- | --- |
| 粉丝数据 | 抖音/小红书/B站后台 | 每日手动录入 |
| 内容数据 | 平台创作者中心 | API自动同步（规划中） |
| 商单数据 | 商务合同+财务系统 | 人工录入 |
| 投放数据 | DOU+/薯条后台 | 每周汇总 |

## 二、核心指标定义

### 2.1 项目级指标

| 指标 | 字段名 | 计算方式 | 目标值 | 预警线 | 风险线 |
| --- | --- | --- | --- | --- | --- |
| 新增签约达人 | new_signed_kols | 当周新签人数 | ≥1名/周 | 0名 | 0名 |
| 头部达人数量 | top_kol_count | S/A级或粉丝≥3万 | ≥8名 | <8名 | <5名 |
| 周均粉丝增长率 | fan_growth_rate | (本周粉丝-上周粉丝)/上周粉丝 | ≥12% | <12% | <5% |
| 商单转化率 | order_conversion_rate | 成交商单数/洽谈数 | ≥15% | <15% | <10% |
| 达人周均变现 | kol_monthly_revenue | 当周总变现/达人数量 | ≥0.7万 | <0.7万 | <0.5万 |

### 2.2 达人级指标

| 指标 | 计算方式 | 用途 |
| --- | --- | --- |
| 粉丝净增 | 当日粉丝数-前日粉丝数 | 增长趋势 |
| 内容产出数 | 当周发布内容数量 | 考核达人义务 |
| 平均播放量 | 当周内容播放量总和/内容数 | 内容质量 |
| 互动率 | (点赞+评论+转发)/播放量 | 粉丝粘性 |
| 完播率 | 完整播放次数/总播放次数 | 内容吸引力 |
| 爆款率 | 播放量>均值3倍的内容数/总内容数 | 爆款能力 |
| 商单数 | 当周完成商单数量 | 变现能力 |

### 2.3 内容级指标

| 指标 | 计算方式 | 达标线 |
| --- | --- | --- |
| 播放量 | 内容被播放次数 | 按达人平均水平评估 |
| 点赞率 | 点赞数/播放量 | ≥3% |
| 评论率 | 评论数/播放量 | ≥0.5% |
| 转发率 | 转发数/播放量 | ≥0.3% |
| 收藏率 | 收藏数/播放量 | ≥1% |
| 粉丝转化率 | 新增粉丝/播放量 | ≥1% |

## 三、看板页面规范

### 3.1 项目总览页

**布局**：

- 顶部：5个KPI卡片（当前值/目标值/状态色块）
- 中部：周趋势折线图（近9周粉丝增长、变现趋势）
- 底部：达人状态矩阵（6达人 × 4核心指标）

**状态色块规则**：

- 🟢 绿色：指标≥目标值
- 🟡 黄色：指标在预警线与目标值之间
- 🔴 红色：指标<预警线（或<风险线）

### 3.2 达人详情页

**必含模块**：

1. 达人基本信息卡片（等级、粉丝量、签约日期）
1. 近30天粉丝增长曲线
1. 内容产出日历（每日发布标记）
1. 内容表现Top10表格
1. 商单记录与收益汇总
1. 风险标签（如有）

### 3.3 内容分析页

**必含模块**：

1. 内容库列表（可按达人/时间/平台筛选）
1. 单条内容数据详情（播放/互动/完播/转化漏斗）
1. 爆款内容标记与复盘链接
1. 评论区关键词云（正面/负面/建议）

## 四、数据更新制度

### 4.1 更新频率

| 数据 | 更新频率 | 负责人 | 截止时间 |
| --- | --- | --- | --- |
| 粉丝数据 | 每日 | 李四 | 每日10:00前 |
| 内容数据 | 内容发布后24/48/72小时 | 李四 | — |
| 商单数据 | 每单更新 | 王五 | 签约/交付/结算当日 |
| 周报数据 | 每周一 | 李四 | 周一上午10:00 |

### 4.2 数据校验

- 每日：检查昨日数据是否完整录入
- 每周：核对平台后台数据与看板数据一致性
- 每月：财务数据与商务数据交叉验证

## 五、风险预警机制

### 5.1 自动预警规则

| 触发条件 | 预警方式 | 通知对象 |
| --- | --- | --- |
| 当周新增签约达人=0名 | 看板标黄+飞书消息 | PM+商务 |
| 头部达人数量<8名 | 看板标黄+飞书消息 | PM |
| 头部达人数量<5名 | 看板标红+飞书消息+邮件 | PM+部门负责人 |
| 粉丝增长率连续2周<12% | 看板标黄 | PM+运营 |
| 粉丝增长率连续2周<5% | 看板标红 | PM+部门负责人 |
| 商单转化率<15% | 看板标黄 | PM+商务 |
| 商单转化率<10% | 看板标红 | PM+部门负责人 |
| 达人周均变现<0.7万 | 看板标黄 | PM+财务 |
| 达人周均变现<0.5万 | 看板标红 | PM+部门负责人 |

### 5.2 预警响应

1. **收到预警** → PM确认数据准确性
1. **确认风险** → 召集相关责任人分析原因
1. **制定对策** → 输出应对方案，明确负责人和截止日期
1. **跟踪闭环** → 每日跟踪对策执行效果，直至指标恢复正常

## 六、周报自动生成

### 6.1 数据源

周报数据直接从看板提取，确保口径一致：

- 数据看板 → 周报"数据看板"部分
- 风险预警记录 → 周报"风险与阻塞"部分
- TODO状态 → 周报"任务完成情况"部分

### 6.2 周报模板字段
```plaintext
1. 数据看板（5项KPI当前值+环比+状态）
2. 风险与阻塞（项目整体状态+触发原因+其他风险）
3. 任务完成情况（已完成 ✅ + 进行中 ⏳）
4. 下周计划（待开始事项）
5. 相关链接（项目文档、会议纪要）

```

## 七、数据源映射（飞书多维表格）

本项目数据看板的数据源统一托管在飞书多维表格（Base），看板页面直接读取 Base 表实时渲染，不再维护独立的 Markdown 看板文档。

### 7.1 Base 表结构

| Base 表名 | 说明 |
| --- | --- |
| 达人档案主表 | 达人基础信息、粉丝数据、商业数据、风险等级 |
| 项目周度KPI | 项目级5项核心KPI的周度汇总 |
| 达人周度快照 | 每位达人每周的数据快照 |
| 内容产出记录 | 单条内容的数据详情（播放/互动/完播） |
| 品牌合作管理 | 品牌/商单的全生命周期管理 |

### 7.2 核心 KPI 与 Base 字段映射

| KPI 指标 | Base 字段 | 计算方式 |
| --- | --- | --- |
| 新增签约达人 | `new_signed_kols` | 当周新签人数 |
| 头部达人数量 | `top_kol_count` | 当周 S/A 级或粉丝≥3万达人数 |
| 周均粉丝增长率 | `avg_fan_growth_rate` | 所有达人当周粉丝增长率均值 |
| 商单转化率 | `order_conversion_rate` | 成交商单数/洽谈数 |
| 达人周均变现 | `avg_revenue_per_kol` | 当周总变现/活跃达人数 |

### 7.3 达人级指标与 Base 字段映射

| 指标 | Base 字段 |
| --- | --- |
| 粉丝净增 | `fan_count`（当前周） vs `initial_fan_count`（签约时） |
| 内容产出数 | `content_count` |
| 平均播放量 | `total_views` / `content_count` |
| 互动率 | `(likes+comments+shares)/views` |
| 完播率 | `avg_completion_rate` |
| 爆款率 | `viral_content_count` / `content_count` |
| 商单数 | `order_count` |

### 7.4 内容级指标与 Base 字段映射

| 指标 | Base 字段 |
| --- | --- |
| 播放量 | `views` |
| 点赞率 | `likes` / `views` |
| 评论率 | `comments` / `views` |
| 转发率 | `shares` / `views` |
| 完播率 | `completion_rate` |
| 是否爆款 | `is_viral` |
| 数据状态 | `data_status` |

### 7.5 品牌合作指标与 Base 字段映射

| 指标 | Base 字段 |
| --- | --- |
| 合作品牌 | `brand_name` |
| 合作阶段 | `stage` |
| 合同金额 | `contract_amount` |
| 实际营收 | `actual_revenue` |
| 结算状态 | `settlement_status` |
| 内容交付进度 | `content_delivered` / `content_required` |
| 满意度评分 | `satisfaction_score` |
| 风险标记 | `risk_flag` + `risk_reason` |

> 注：数据看板不再维护独立的 Markdown 看板文档，所有看板数据均来自飞书 Base 实时查询。
