## 二、提示词工程（Prompt Engineering）

### 2.1 脚本生成提示词模板
```plaintext
角色：你是一位资深短视频内容策划，擅长[美妆/穿搭/美食]赛道。
任务：为[达人名称/人设]撰写一条[时长]的短视频脚本。
要求：
- 目标平台：[抖音/小红书/B站]
- 选题方向：[具体选题]
- 开头3秒必须有强钩子
- 口播风格：[专业/亲切/幽默/毒舌]
- 结尾包含CTA（关注/点赞/评论引导）
- 标注每段的预计时长

输出格式：
【标题】
【钩子】（0-3秒）
【正文】（分段标注时长）
【结尾CTA】

```

### 2.2 优化提示词示例

**初版输出不满意时**：
```plaintext
优化要求：
- 钩子再强一些，使用反常识或强对比
- 减少专业术语，口语化表达
- 增加1个互动提问点
- 总时长控制在45秒以内

```

## 三、AI 脚本工作流
```plaintext
1. 选题输入 → AI生成初稿（3-5个版本）
2. 人工筛选 → 选择最优版本或拼接
3. 合规检查 → 对照禁区清单审核
4. 达人适配 → 根据达人风格调整语气
5. 最终定稿 → 提交审核流程

```

**效率对比**：

- 纯人工：1条脚本30-60分钟
- AI辅助：1条脚本10-15分钟
- 提效幅度：50%-70%

## 四、注意事项

### 4.1 AI 脚本常见风险

| 风险 | 表现 | 对策 |
| --- | --- | --- |
| 事实错误 | AI编造产品成分/功效 | 必须人工核对产品信息 |
| 极限词违规 | "最好""第一""绝对" | 人工替换为合规表达 |
| 同质化 | 与平台现有内容雷同 | 加入达人个人经历/独特视角 |
| 缺乏情感 | 读起来像说明书 | 增加场景化描述和情绪词 |

### 4.2 人机分工原则

- **AI 擅长**：框架搭建、多版本生成、标题优化、格式整理
- **人 必须做**：事实核查、风格适配、合规审核、情感注入

## 五、实战案例：美妆测评脚本

**AI 初稿**（经优化后）：
```plaintext
【标题】30块的粉底居然比300块的还遮瑕？
【钩子】（0-3秒）"这瓶30块的粉底，遮瑕力吊打我手里这瓶300的"
【正文】
（3-8秒）左边脸涂30块的，右边涂300块的
（8-18秒）近镜头展示遮瑕对比，斑点/痘印覆盖情况
（18-23秒）4小时后妆效对比，暗沉/脱妆情况
（23-28秒）总结：贵的在细腻度上胜出，但遮瑕确实不如便宜的
【结尾CTA】"你用过什么平价好物？评论区反向安利我"

```

**人工优化点**：加入达人真实使用经历，替换AI生成的"吊打"为更口语化的"居然比...还..."
