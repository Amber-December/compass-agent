# Interactive Math Courseware（转盘密码锁范式）

用于把“数学规则”稳定转成“可交互课件”。

> 本 skill 附带可复用代码：`scripts/lockMath.ts`、`scripts/puzzlePresets.ts`。遇到类似课件时优先复用这些脚本，而不是重复手写核心数学逻辑。

## 1) 先定数学模型，再做动画

先明确三件事：
1. 状态空间（如刻度 0~20，共 21 格）
2. 运算规则（+/- 对应刻度增减）
3. 显示规则（指针固定 or 表盘固定）

> 原则：动画只是状态变化的可视化，不能反过来主导数学逻辑。

## 2) 统一方向语义（避免“顺时针到底加还是减”的混乱）

- 在核心逻辑文件顶部写清楚约定。
- 代码内保持单一真相：UI 文案、内部 direction、旋转角正负必须一致。
- 提供从“旋转角 -> 当前刻度”的单独函数，供全局复用。

## 3) 必做取模归一化

对所有刻度结果做 normalize（例如 0~20）：
- 支持跨圈旋转
- 支持负向输入
- 避免边界 bug

## 4) 交互节奏：单步推进优先

练习页优先采用“下一步 -> 演示一步”节奏：
- 每次只暴露一个认知动作
- 记录累计状态（如累计旋转角）用于断点续播
- 演示完成后再进入答题，避免同时处理多个任务

## 5) 动画参数建议（Framer Motion）

- 旋转主体：spring（stiffness 50~90 / damping 14~22）
- 页面切换：AnimatePresence + mode="wait"
- 反馈动画：
  - 正确：发光 + 缩放 + 宝箱开盖
  - 错误：短促抖动（输入框）

> 优先“可读性与跟手感”，其次才是炫技。

## 6) 页面信息密度策略

- 交互教学页：尽量单屏完成（不滚动或轻滚动）
- 总结复盘页：允许滚动，容纳知识卡片和步骤表

## 7) 复用组件清单（建议）

- `CombinationLock`（核心状态与播放控制）
- `StepIndicator`（进度可视化）
- `NavigationControls`（教学流程导航）
- `TreasureChest`（结果反馈仪式感）
- `PageTransition`（章节切换）

## 8) 验收清单

上线前至少核对：
- [ ] 数学结果与动画展示 100% 一致
- [ ] 顺/逆时针文案与内部方向一致
- [ ] 跨圈样例（如 -15）结果正确
- [ ] 单步演示不会重置到初始状态
- [ ] 正误反馈明确且节奏不打断教学
- [ ] 手机端单手操作无阻碍

## 9) 适用任务模板

当用户要求“做一个互动数学课件”，默认输出应包含：
1. 数学规则映射表（规则 -> 状态 -> UI）
2. 页面流程（引入 / 探索 / 练习1 / 练习2 / 总结）
3. 关键组件与 Props 设计
4. 动画参数建议
5. 易错点与测试用例（至少含一个循环取模案例）
