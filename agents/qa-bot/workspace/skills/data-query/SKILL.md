# data-query Skill

> 结构化数据智能查询 — 从飞书 Base 中精准检索记录，支持单条查询、跨周对比、聚合统计（带权限隔离）

## 归属

- **Agent**: qa-bot
- **路径**: `agents/qa-bot/workspace/skills/data-query/`
- **工具**: `agents/qa-bot/workspace/tools/data_query.py`

## 触发条件

- 用户意图被识别为 **数据查询**
- 典型问法：
  - 单条查询：`"K001 Week 2 粉丝数"`、`"苏小棠本周收入"`
  - 跨周对比：`"K001 Week 1 和 Week 2 粉丝对比"`、`"苏小棠粉丝增长了多少"`
  - 聚合统计：`"所有达人的周收入"`、`"Week 3 各达人粉丝数"`
  - 复合查询：`"头部达人平均完播率"`、`"K003 所有周的播放量汇总"`

## 核心能力

### 1. 模糊匹配 — 默认优先检索本项目数据

当用户在项目群中提问时，如果问题**未明确指定项目名称**或**未指定达人编号/昵称**，则**默认优先检索当前 scope 对应的项目数据**。

**模糊查询识别规则**：

| 模糊类型 | 示例 | 默认指向 |
|----------|------|----------|
| 省略项目名称 | "本周 GMV 多少"、"转化率多少" | 当前 scope 对应的项目 |
| 使用代词 | "咱们项目"、"我们这周" | 当前 scope 对应的项目 |
| 仅问指标 | "活跃达人数量"、"头部达人有几个" | 当前 scope 对应的项目 |
| 未指定达人 | "达人周均变现"（未提 K001/苏小棠） | 当前 scope 对应的项目级 KPI |
| 跨项目明确指定 | "直播电商 Week 8 GMV" | 按明确项目名称检索，不受默认规则影响 |

**默认本项目规则**：
- 当 scope 为具体项目（非 `None` / `dept`）时，优先检索本项目数据
- 如果当前 scope 内无匹配，再提示用户补充项目信息
- 数据查询默认聚合到项目级，不自动跨项目汇总

**示例**（scope = `kol-incubation`）：
- 用户问："达人周均变现多少" → 默认读取 kol-incubation 的 Week 9 项目 KPI
- 用户问："活跃达人数量" → 默认读取 kol-incubation 的 Week 9 项目 KPI
- 用户问："直播电商转化率" → 明确指定跨项目，按跨项目规则处理

---

### 2. 智能查询解析

工具自动从自然语言问题中提取查询参数，无需用户记忆字段名：

| 提取维度 | 识别方式 | 示例 |
|---------|---------|------|
| **达人编号** | `K001`、`K002` 格式 | `"K001 的粉丝数"` |
| **达人昵称** | 2-4 字中文名称 | `"苏小棠 Week 2 收入"` |
| **周次** | `Week 2`、`第2周`、`周2` | `"Week 3 数据"` |
| **日期范围** | `2026-03-09 至 2026-03-15` | 直接匹配日期字段 |
| **指标** | 粉丝数、播放量、收入、完播率、增长率等 | `"平均完播率"` |
| **数据表** | 达人周度快照、内容产出、品牌合作、项目KPI 等 | `"内容产出数据"` |
| **对比意图** | `对比`、`增长`、`变化`、`环比` | `"Week 1 和 Week 2 对比"` |
| **聚合意图** | `所有`、`全部`、`平均`、`汇总` | `"所有达人的周收入"` |

### 3. 权限检查

数据查询受 `scope` 限制：

| scope | 可访问数据 |
|-------|-----------|
| `None` | 所有项目数据（兜底/管理员） |
| `dept` | 部门级聚合数据，不暴露单项目明细 |
| `kol-incubation` | 仅垂类达人孵化项目数据 |
| `brand-ops` | 仅品牌代运营项目数据 |
| ... | 对应项目数据 |

**跨项目数据拒答（关键）**：

数据查询受 `scope` 严格限制，**不允许通过数据查询获取跨项目敏感数据**：

| 场景 | 处理方式 |
|------|----------|
| 查询当前 scope 对应项目的数据 | 正常回答 |
| 查询跨项目的数据（明确指定其他项目） | **必须拒答** |
| 问题模糊无法判断项目归属 | 默认按当前 scope 查询；如果检索不到，提示用户补充项目名称 |

**敏感数据定义**：所有来自飞书 Base 的数值型数据，包括但不限于：
- 金额类：GMV、营收、周均变现、客单价
- 比率类：转化率、ROI、完播率、粉丝增长率、复购率
- 数量类：活跃达人数量、签约数量、粉丝数、播放量
- 绩效类：达人收入、客户满意度评分、技术故障率

**拒答话术**：
```
⚠️ 跨项目敏感数据未开放，如需 [项目名] 的详情，请到该项目群内询问。
```

**权限规则**：
- 查询前检查 scope 是否允许访问对应项目
- 无权限时返回：`"该项目数据不在您的访问范围内。"`

### 4. 数据表说明

常用数据表：

| 数据表 | 内容 | 关键字段 |
|------|------|---------|
| 达人周度快照 | 达人每周核心指标 | 达人编号、昵称、周次、粉丝数、总播放量、周收入、平均完播率、粉丝增长率、等级 |

## 查询链路

```
用户提问
    └── 意图识别（intent skill）
        └── 数据查询 → data-query skill
            ├── 接收 scope 参数
            ├── 调用 data_query.query_data(question, scope)
            │   ├── 解析问题：提取达人编号/昵称、周次、指标、对比/聚合意图
            │   ├── 按条件过滤记录
            │   ├── 如为对比查询：自动计算变化量、变化率
            │   ├── 如为聚合查询：自动计算平均/最高/最低
            │   └── 返回结构化 Markdown 表格 + 分析结果 + 对应 Base 链接
            ├── 判断返回结果
            │   ├── 有数据 → 构建 LLM Prompt（数据表格 + 分析结论 + 问题）
            │   └── 无数据 → 降级为文档查询（调用 wiki_query.query_wiki）
            │       ├── 检索相关 SOP、规范、项目文档
            │       └── 返回文档链接（飞书 Wiki 链接）
            └── 调用 OpenClaw LLM 生成自然语言回答
```

## 工具使用

### 调用 data_query.py

```python
from tools.data_query import query_data

result = query_data("K001 Week 2 粉丝数", scope="kol-incubation")
# result = {
#     "records": [{"达人编号": "K001", "达人昵称": "苏小棠", "周次": 2, "粉丝数": 55000, ...}],
#     "context": "| 达人编号 | ... |\n| --- | ... |\n| K001 | ... |",  # Markdown 表格
#     "analysis": "苏小棠 的 粉丝数 跨周变化：\n- Week 1 → Week 2: 30,000 → 55,000 (变化: +25,000, +83.33%)",
#     "base_url": "https://ncnkdep1f4r7.feishu.cn/base/LTLNbBlGTaB4mNs6by0cvhPhnib",
#     "params": {"kol_id": "K001", "weeks": [2], "metrics": ["粉丝数"], "compare": False, ...},
#     "total_records": 1,
# }
```

### CLI 用法

```bash
# 单条查询
python tools/data_query.py "K001 Week 2 粉丝数" --scope kol-incubation --json

# 跨周对比
python tools/data_query.py "苏小棠 Week 1 和 Week 2 粉丝对比" --scope kol-incubation --json

# 聚合统计
python tools/data_query.py "所有达人的周收入" --scope kol-incubation --json

# 复合条件
python tools/data_query.py "Week 3 头部达人的平均完播率" --scope kol-incubation --json
```

## 回答生成规范（强制）

LLM 基于工具的返回生成回答时：

**绝对禁止**：
- 输出思考过程、推理步骤、分析过程
- 说"让我查一下"、"根据数据分析"等中间状态
- 分多条消息输出，只输出一条最终消息
- **暴露本地文件路径**（如 `workspace/knowledge/data/...`、`*.json`）
- **暴露本地检索过程**（如"扫描文件"、"读取 JSON"、"过滤记录"）

**必须包含**：
- 具体数值（直接引用 `context` 表格中的数据）
- 分析结论（引用 `analysis` 中的计算结果）
- **数据来源**：飞书 Base 链接（来自 `base_url`），标注为"飞书 Base"
- 时间范围（数据对应的周次或日期）
- **只输出一条消息**，就是最终答案本身，严禁任何元评论

**降级规则**：
- 数据查询无结果时，**自动调用 wiki_query** 检索相关文档
- 返回相关 SOP、项目文档、数据说明的链接
- 明确告知最新数据的时间范围，不编造、不外推
- 文档链接必须是飞书 Wiki 链接，不使用本地路径

**权限不足时**：
```markdown
该项目数据不在您的访问范围内。

- 在项目群内查询本项目数据
- 联系 wiki-manager 管理员申请跨项目权限
```

## 红线

- **严禁输出思考过程** — 只输出一条最终消息，禁止展示推理步骤、分析过程、"让我想想"等中间状态
- **严禁暴露本地文件路径** — 数据来源只能使用飞书 Base 链接，禁止使用 `workspace/`、`*.json` 等本地路径
- **严禁暴露检索过程** — 禁止描述"扫描文件"、"读取 JSON"、"过滤记录"等内部操作
- **数据必须来自飞书 Base**，禁止编造
- **无数据时不预测**，但**必须降级为文档查询**，返回相关 SOP/规范的飞书 Wiki 链接，不能只说一句"暂无数据"
- **跨项目数据需验证权限**，敏感 KPI 不随意回答
- **scope 检查必须在数据读取前执行**
