# query Skill

> 知识库检索与回答 — 基于飞书 Wiki 知识底座回答用户问题（带权限隔离）

## 归属

- **Agent**: qa-bot
- **路径**: `agents/qa-bot/workspace/skills/query/`
- **工具**: `agents/qa-bot/workspace/tools/wiki_query.py`

## 触发条件

- 用户意图被识别为 **知识查询** 或 **个人介绍**
- 典型问法："什么是..."、"如何..."、"为什么..."、"介绍下..."、"xxx 是什么"

## 核心能力

### 1. 模糊匹配 — 默认优先检索本项目

当用户在项目群中提问时，如果问题**未明确指定项目名称**（省略了"垂类达人孵化"、"直播电商"等），则**默认优先检索当前 scope 对应的项目文档**。

**模糊查询识别规则**：

| 模糊类型 | 示例 | 默认指向 |
|----------|------|----------|
| 省略项目名称 | "咱们项目的 PM 是谁"、"本周数据怎么样" | 当前 scope 对应的项目 |
| 使用代词 | "我们项目"、"本项目"、"群里这个项目" | 当前 scope 对应的项目 |
| 仅问指标 | "达人周均变现多少"、"商单转化率多少" | 当前 scope 对应的项目 |
| 仅问 SOP | "达人筛选标准是什么" | 当前 scope 对应的项目（如 kol-incubation） |
| 跨项目明确指定 | "直播电商的 PM 是谁" | 按明确项目名称检索，不受默认规则影响 |

**默认本项目规则**：
- 当 scope 为具体项目（非 `None` / `dept`）时，优先检索本项目对应文档
- 如果当前 scope 内无匹配，再扩展到公共知识、部门知识、通用概念
- 回答时默认使用本项目数据，无需用户重复指定项目名称

**示例**（scope = `kol-incubation`）：
- 用户问："PM 是谁" → 默认回答垂类达人孵化的 PM（李小明）
- 用户问："本周达人周均变现" → 默认读取 kol-incubation 的 Week 9 数据
- 用户问："直播电商的 PM" → 明确指定跨项目，按跨项目规则处理

---

### 2. 权限隔离

qa-bot 接收当前会话的 `scope` 参数，查询时只返回该 scope 允许访问的内容。

| scope | 可访问范围 |
|-------|-----------|
| `None` | 公共知识 + 部门知识 + 实体百科 + 通用概念 + 所有项目知识（需验证） |
| `dept` | 公共知识 + 部门知识 + 实体百科 + 通用概念 |
| `kol-incubation` | 公共知识 + 部门知识 + **垂类达人孵化**项目知识 + 实体百科 + 通用概念 |
| `brand-ops` | 公共知识 + 部门知识 + **品牌代运营**项目知识 + 实体百科 + 通用概念 |
| `live-commerce` | 公共知识 + 部门知识 + **直播电商**项目知识 + 实体百科 + 通用概念 |
| `short-video` | 公共知识 + 部门知识 + **短视频内容矩阵**项目知识 + 实体百科 + 通用概念 |
| `private-domain` | 公共知识 + 部门知识 + **达人私域运营**项目知识 + 实体百科 + 通用概念 |
| `virtual-ip` | 公共知识 + 部门知识 + **虚拟IP孵化**项目知识 + 实体百科 + 通用概念 |

**跨项目敏感信息过滤（关键）**：

虽然实体百科（entities/）对所有 scope 开放，但其中的**详细 KPI 数值**属于敏感数据，必须按以下规则过滤：

| 信息类型 | 示例 | 跨项目（非本项目）scope | 处理方式 |
|----------|------|----------------------|----------|
| **项目基本信息** | PM 姓名、成员名单、启动时间 | 正常回答 | 属于部门公开信息 |
| **项目状态概览** | 🟢正常/🟡预警/🔴风险 | 正常回答 | 属于部门看板公开信息 |
| **具体 KPI 数值** | GMV 487,500、ROI 1:2.5、转化率 4.2% | **必须拒答** | 属于敏感数据 |
| **周度/月度明细** | Week 8 粉丝增长率 20.19% | **必须拒答** | 属于敏感数据 |
| **达人/客户具体数据** | 苏小棠 Week 2 收入 42,000 | **必须拒答** | 属于敏感数据 |
| **整改进度细节** | 整改方案执行情况 | **必须拒答** | 属于敏感数据 |

**过滤规则**：
1. 识别用户问题是否涉及**具体数值**（金额、百分比、数量、比率）
2. 如果涉及具体数值，且目标项目**不是当前 scope 对应的项目**，则拒绝回答
3. 拒绝时统一话术："⚠️ 跨项目敏感数据未开放，如需 [项目名] 的详情，请到该项目群内询问。"

**权限规则**：
- 关键词匹配和图邻居扩展都受权限过滤
- 用户询问无权限项目内容时，返回："该项目数据不在您的访问范围内，您可以联系项目负责人或 wiki-manager 管理员申请权限。"

## 查询链路（配合 OpenClaw LLM）

```
用户提问
    └── 意图识别（intent skill）
        └── 知识查询 → query skill
            ├── 接收 scope 参数
            ├── 调用 wiki_query.query_wiki(question, scope)
            │   ├── 匹配相关页面（受权限过滤）
            │   ├── 图邻居扩展（受权限过滤）
            │   └── 返回页面内容上下文 + 飞书 Wiki 链接
            ├── 构建 LLM Prompt
            │   └── 包含：页面内容 + 问题 + 回答要求
            └── 调用 OpenClaw LLM 生成回答
                └── 返回结构化回答
```

## 工具使用

### 调用 wiki_query.py

```python
# 检索知识库页面
result = query_wiki(question, scope=scope)
# result = {
#     "pages": [...],
#     "context": "页面内容上下文（供 LLM 使用）",
#     "sources": [{"title": "...", "lark_url": "https://ncnkdep1f4r7.feishu.cn/wiki/..."}],
#     "total_pages": N
# }
```

### 调用 OpenClaw LLM

```python
# 构建 Prompt
prompt = f"""你是 Compass（司南）MCN 事业部的知识问答助手。请基于以下知识库页面直接给出最终答案。

**绝对禁止**：
- 输出思考过程、推理步骤、分析过程
- 说"让我想想"、"我来查一下"、"根据资料分析"等中间状态
- 分多条消息输出，只输出一条最终消息
- 暴露本地文件路径或检索过程

回答要求：
1. **只输出一条消息**，就是最终答案本身，严禁任何元评论
2. 使用 Markdown 格式，结构清晰
3. 关键事实必须标注来源，使用 [[PageName]] 的 wikilink 语法
4. 如果涉及飞书原始文档，在 Sources 部分提供链接
5. 如果知识库中没有相关信息，明确说明"根据现有资料，暂未找到相关信息"
6. 不要编造数据中不存在的信息
7. 如果用户询问的内容涉及无权限的项目数据，礼貌说明"该项目数据不在您的访问范围内"

知识库页面：
{result["context"]}

问题：{question}

请直接给出结构化的最终回答，包含标题、要点和 Sources 引用部分。不要输出任何思考过程。"""

# 使用 OpenClaw 内置 LLM 生成回答
answer = call_openclaw_llm(prompt)
```

## 回答生成规范

**必须包含**：
- 直接回答用户问题的核心信息
- `[[PageName]]` 引用来源（至少 1 个，不超过 3 个）
- 如果涉及多个观点，说明来源差异
- **数据来源**：飞书 Wiki 链接（来自 `sources` 中的 `lark_url`）

**禁止**：
- 编造知识库中没有的信息
- 引用不存在的页面
- 暴露 raw_lark/ 原始内容
- 跨 scope 泄露无权限内容
- **暴露本地文件路径**（如 `workspace/`、`*.md`、`*.json`）
- **暴露检索过程**（如"读取文件"、"扫描目录"、"匹配页面"）

## 输出格式

```markdown
**回答**：（核心结论，2-4 句话）

**详细说明**：
- 要点 1
- 要点 2

**来源**：
- [[PageName1]] — 具体引用内容
- [[PageName2]] — 具体引用内容

**飞书原文**：
- [PageName1](https://ncnkdep1f4r7.feishu.cn/wiki/xxxx)
- [PageName2](https://ncnkdep1f4r7.feishu.cn/wiki/yyyy)
```

**权限不足时的输出**：
```markdown
该项目数据不在您的访问范围内。

您可以：
- 联系项目负责人申请权限
- 联系 wiki-manager 管理员调整 scope 配置
```

## 缺失处理

如果知识库中没有匹配内容：
- 明确告知 "当前知识库中没有关于 xxx 的信息"
- 建议用户："您可以尝试更新知识库，或联系 wiki-manager 管理员"

## 红线

- **严禁输出思考过程** — 只输出一条最终消息，禁止展示推理步骤、分析过程、"让我想想"等中间状态
- **严禁暴露本地文件路径** — 数据来源只能使用飞书 Wiki 链接，禁止使用 `workspace/`、`*.md` 等本地路径
- **严禁暴露检索过程** — 禁止描述"读取文件"、"扫描目录"、"匹配页面"等内部操作
- **必须引用来源**，不允许无来源的断言
- **不编造**，不确定时明确说明
- **不跨 scope 泄露**：projects 级文档内容不回答给无权限用户
- **权限过滤必须在检索阶段执行**，不能等生成答案后才过滤
